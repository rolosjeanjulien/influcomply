# Exigences — Publication Compliance Scanner (SCN)

> Module cœur de la plateforme. Analyse les publications des créateurs pour détecter les non-conformités réglementaires.

## Exigences

| ID | Titre | Exigence | Type | Prio | Source | Vérification | Rationale |
|----|-------|----------|------|------|--------|-------------|-----------|
| REQ-SCN-001 | Connexion comptes sociaux | Le système DOIT permettre à un créateur de connecter ses comptes Instagram, TikTok et YouTube via les protocoles OAuth2 de chaque plateforme. | F | P0 | PRD §4.1 | Test d'intégration | Les API officielles sont le canal primaire d'acquisition de données de publications. |
| REQ-SCN-002 | Acquisition multi-mode | Le système DOIT fournir au minimum trois modes d'acquisition de données de publication : (a) API OAuth, (b) extension navigateur, (c) import manuel (URL ou capture d'écran). | F | P0 | PRD §4.1 critique | Test fonctionnel | Réduit la dépendance aux API de plateformes dont l'accès peut être restreint. |
| REQ-SCN-003 | Détection mention publicitaire | Le système DOIT détecter la présence ou l'absence des mentions obligatoires « Publicité » ou « Collaboration commerciale » dans le texte, les légendes, les sous-titres, et les premiers frames d'une vidéo, conformément à l'art. 5 de la loi n° 2023-451. | F | P0 | Loi 2023-451 art.5 | Test NLP + règles | Obligation légale ; infraction la plus fréquemment constatée par la DGCCRF (50% des contrôles). |
| REQ-SCN-004 | Détection produits interdits | Le système DOIT classifier le contenu d'une publication et alerter si elle fait la promotion de catégories interdites : chirurgie esthétique, nicotine, abstention thérapeutique, certains produits financiers (crypto non-régulée), paris sportifs pour mineurs. | F | P0 | Loi 2023-451 art.4 | Test classification + golden dataset | Violation passible de 300 000 € d'amende et 2 ans d'emprisonnement. |
| REQ-SCN-005 | Score de conformité | Le système DOIT calculer un score de conformité entre 0 et 100 pour chaque publication scannée, décomposé en sous-scores par critère vérifiable. | F | P0 | PRD §4.1 | Test unitaire calcul | Permet une vision quantifiée et actionnable du niveau de conformité. |
| REQ-SCN-006 | Alertes temps réel | Le système DOIT notifier l'utilisateur (push, email, ou notification in-app) dans un délai maximal de 15 minutes après la détection d'une non-conformité sur une publication scannée. | F | P0 | PRD §4.1 | Test de latence | Permet une correction avant que la publication ne génère de l'engagement significatif. |
| REQ-SCN-007 | Suggestions correctives | Le système DEVRAIT proposer pour chaque non-conformité détectée une suggestion de correction textuelle exploitable (mention à ajouter, formulation recommandée). | F | P1 | PRD §4.1 | Test qualitatif UX | Réduit le temps de correction et la friction utilisateur. |
| REQ-SCN-008 | Détection images retouchées | Le système DEVRAIT détecter, via l'analyse des métadonnées EXIF et des heuristiques d'image, les contenus susceptibles d'avoir été retouchés et pour lesquels la mention obligatoire est absente. | F | P1 | Loi 2023-451 art.5 | Test avec jeu de données annotées | Obligation légale d'indiquer les images retouchées modifiant la silhouette. |
| REQ-SCN-009 | Historique de scans | Le système DOIT conserver un historique horodaté de tous les scans effectués et de leurs résultats pendant une durée minimale de 3 ans. | F | P0 | RGPD + preuve de conformité | Test d'archivage | Nécessaire comme preuve en cas de contrôle DGCCRF. |
| REQ-SCN-010 | Scan batch multi-publications | Le système DEVRAIT permettre le scan simultané de l'ensemble des publications d'un créateur sur une période donnée (scan rétroactif). | F | P1 | Persona agence | Test de charge | Les agences doivent pouvoir auditer un créateur avant activation. |
